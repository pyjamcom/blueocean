# Data Governance

## Принципы
- Минимум данных, без PII.
- Случайные идентификаторы комнат/игроков.

## Retention
- Оперативные логи/сессии: до 30 дней.
- Агрегированные метрики: до 12 месяцев.
- Ассеты: до отзыва лицензии.

## Deletion requests
- Удаление по запросу: удалить события по user_id/crew_code за 14 дней с момента запроса.
- Ключи удаления: user_id, crew_code, session_id (если доступно).
- Минимизация: хранить только агрегаты после удаления (без идентификаторов).
